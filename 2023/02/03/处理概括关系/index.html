    重构列表-处理概括关系(继承关系)

    有一批重构手法专门用来处理类的概括关系（generalization，即继承关系），其中主要是将函数上下移动于继承体系之中。

    1. Pull Up Field（字段上移）

    两个子类拥有相同的字段。

    将该字段移至超类。

    class Empolyee{}

class Salesman extends Employee {
  private String name;
}
class Engineer extends Employee {
  private String name;
}

    class Empolyee{
    protected String name;
}
class Salesman extends Employee {}
class Engineer extends Employee {}

    动机

    • 去除了重复的数据声明
    • 该字段的行为从子类移动到超类，从而去除重复行为

    2. Pull Up Method（函数上移）

    有些函数，在各个子类中产生完全相同的结果。

    将函数移至超类。

    class Empolyee{}

class Salesman extends Employee {
  private String getName(){}
}
class Engineer extends Employee {
  private String getName(){}
}

    class Empolyee{
    protected String getName(){}
}
class Salesman extends Employee {}
class Engineer extends Employee {}

    动机

    • 避免行为重复

    3. Pull Up Constructor Body（构造函数本体上移）

    你在各个子类中拥有一些构造函数，它们的本体几乎完全一致。

    在超类中新建一个构造函数，并在子类构造函数中调用它

    class Manager extends Employee{
  public Manager(String name,String id,int grade) {
    _name = name;
    _id = id;
    _grade = grade;
  }
}

    
class SalesManager extends Manager {
   public SalesManager(String name,String id,int grade) {
    super(name,id,grade);
  }
}

    动机

    • 如果看见各个子类中的函数有共同行为，第一个念头应该是将共同行为提炼到一个独立函数中，然后将这个函数提升到超类。

    4. Push Down Method（函数下移）

    超类中的某个函数只与部分（而非全部）子类有关。

    将这个函数移到相关的那些子类去。

    class Employee {
  protected void getQuota(){}
}
class Salesman extends Employee {}
class Engineer extends Employee {}

    class Employee {}
class Salesman extends Employee {
    private void getQuota(){}
}
class Engineer extends Employee {}

    动机

    • 当有必要把某些行为从超类移至特定的子类时。

    5. Push Down Field（字段下移）

    超类中的某个字段只被部分（而非全部）子类用到。

    将这个字段移到需要它的那些子类去。

    class Employee {
  protected String quota;
}
class Salesman extends Employee {}
class Engineer extends Employee {}


    class Employee {}
class Salesman extends Employee {
    private String quota;
}
class Engineer extends Employee {}

    动机

    • 如果只有某些（而非全部）子类需要超类内的一个字段，你可以使用本项重构。

    6. Extract Subclass（提炼子类）

    类中的某些特性只被某些（而非全部）实例用到。

    新建一个子类，将上面所说的那一部分特性移到子类中。

    class JobItem {
  double getTotalPrice(){}
  double getUnitPrice(){}
  Employee getEmployee(){}
}

    class JobItem {
  double getTotalPrice(){}
  double getUnitPrice(){}
  Employee getEmployee(){}
}

class LaborItem extends JobItem {
  double getUnitPrice(){}
  Employee getEmployee(){}
}

    动机

    • 发现类中的某些行为只被一部分实例用到，其他实例不需要它们。

    7. Extract Superclass（提炼超类）

    两个类有相似特性。

    为这两个类建立一个超类，将相同特性移至超类。

    class Department {
  String getTotalAnnualCost() {}
  String getName() {}
  int getHeadCount() {}
}

class Employee {
  String getTotalAnnualCost() {}
  String getName() {}
  int getId() {}
}

    class Party {
  String getTotalAnnualCost() {}
  String getName() {}
}
class Department extends Party{
  int getHeadCount() {}
}
class Employee extends Party{
  int getId() {}
}

    动机

    • 重复代码是系统中最糟糕的东西之一。
    • 两个类以相同的方式做类似的事情，或者以不同的方式做类似的事情。

    8. Extract Interface （提炼接口）

    若干客户使用类接口中的同一子集，或者两个类的接口有部分相同。

    将相同的子集提炼到一个独立接口中。

    class Employee {
  double getRate(){}
  boolean hasSpecialSkill(){}
  String getName() {}
  String getDepartment(){}
}

    interface Billable {
  double getRate();
  boolean hasSpecialSkill();
} 

class Employee implements Billable {
  double getRate(){}
  boolean hasSpecialSkill(){}
  String getName() {}
  String getDepartment(){}
}

    动机

    • 将真正用到的这部分责任分离出来通常很有意义，因为这样可以使系统的用法更清晰，同时也更容易看清系统的责任划分。

    9. Collapse Hierarchy（折叠继承体系）

    超类和子类之间无太大区别。

    将它们合为一体。

    class Employee {}
class Salesman extends Employee {}

    class Employee {}

    动机

    • 发现某个子类并未带来该有的价值，因此需要把超类与子类合并起来。

    10. Form Template Method（塑造模板函数）

    你有一些子类，其中相应的某些函数以相同顺序执行类似的操作，但各个操作的细节上有所不同。

    将这些操作分别放进独立函数中，并保持它们都有相同的签名，于是原函数也就变得相同了。然后将原函数上移至超类。

    class Site {}

class ResidentialSite extends Site {
  String getBillableAmount() {
    double base = _units * _rate;
    double tax = base * Site.TAX_RATE;
    return base + tax;
  }
}

class LifelineSite extends Site {
  String getBillableAmount() {
    double base = _units * _rate * 0.5;
    double tax = base * Site.TAX_RATE * 0.2;
    return base + tax;
  }  
}

    abstract class Site {
  double getBillableAmount(){
    return getBaseAmount() + getTaxAmount();
  }
  abstract double getBaseAmount(){}
  abstract double getTaxAmount(){}
}

class ResidentialSite extends Site {
  double getBaseAmount(){
    return _units * _rate;
  }
  double getTaxAmount(){
    return base * Site.TAX_RATE;
  }
}

class LifelineSite extends Site {
   double getBaseAmount(){
    return _units * _rate * 0.5;
  }
  double getTaxAmount(){
    return base * Site.TAX_RATE * 0.2;
  }
}

    动机

    • 两个函数以相同顺序执行大致相近的操作，但是各操作不完全相同。

    11. Replace Inheritance with Delegation（以委托取代继承）

    某个子类只使用超类接口中的一部分，或是根本不需要继承而来的数据。

    在子类中新建一个字段用以保存超类；

    调整子类函数，令它改而委托超类；

    然后去掉两者之间的继承关系。

    class Vector {
  boolean isEmpty(){}
}

class Stack extends Vector {  }

    class Stack {
  private Vector _vector = new Vector();
  
  public boolean isEmpty(){
    return _vector.isEmpty();
  }
}

    动机

    • 以委托取代继承，可以更清楚表明：只需要受托类的一部分功能。

    12. Replace Delegation with Inheritance （以继承取代委托）

    你在两个类之间使用委托关系，并经常为整个接口编写许多极简单的委托函数。

    让委托类继承受托类。

    class Person {
  String getName(){}
}

class Employee {
  private Person person = new Person();
  
  String getName() {
    return person.getName();
  }
}

    class Employee extends Person {
  String getName(){}
}

    动机

    • 如果发现需要使用受委托类中的所有函数，并且费了很大力气编写所有极简的委托函数。

    告诫

    • 并没有使用受托类的所有函数，那么就不使用本重构
    • 受托对象被不止一个其他对象共享，而且受托对象是可变的。
