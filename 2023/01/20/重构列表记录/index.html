    几乎所有时刻，问题都源于Long Method(过长函数)

    Extract Method(提炼函数)

    void printOwing(double amount) {
        printBanner();
        
        //print details
        System.out.println("....");
        System.out.println("...." + amount);
}

    将“//print details”这段代码放到独立函数当中，并让函数名解释用途。

    void printOwing(double amount) {
        printBanner();
        //print details
        printDetails(amount);
}

void printDetails(double amount){
        System.out.println("....");
        System.out.println("...." + amount);
}

    动机

    简短且命名良好的函数：

    • 颗粒度小，被复用的机会就更大
    • 使高层函数读起来像一系列注释
    • 覆写更容易

    Inline Method（内联函数）

    在函数调用点插入函数本体，然后移除该函数

    int getRating() {
    return (moreThanFiveLateDeliveries()) ? 2 : 1;
}

boolean moreThanFiveLateDeliveries(){
    return _numberOfLateDeliveries > 5;
}

    int getRating() {
    return _numberOfLateDeliveries > 5 ? 2 : 1;
}

    动机

    • 非必要的间接性让人不舒服
    • 如果有一群不合理的函数，可以先将它们内联到一个大型函数当中，再从中提炼出合理的小函数。
    • 使用了太多间接层。找出有用的间接层，将无用的间接层去除。

    注意

    • 如果具备多态特性，子类重写了该函数，不要将函数进行内联

    Inline Temp（内联临时变量）

    将所有对该变量的引用动作，替换为对它赋值的那个表达式自身。

    double basePrice = anOrder.basePrice();
return (basePrice > 1000)

    return anOrder.basePrice() > 1000

    动机

    • 某个临时变量被赋予某个函数的返回值

    Replace Temp with Query（以查询取代临时变量）

    将这个临时变量的引用点替换为对新函数的调用。

    double basePrice = _quantity * _itemPrice;
if (basePrice > 1000)
    return basePrice * 0.95;
else 
    return basePrice * 0.98;

    if (basePrice() > 1000)
    return basePrice() * 0.95;
else 
    return basePrice() * 0.98;

double basePrice(){
    return _quantity * _itemPrice;
}

    动机

    • 临时变量替换为一个查询，那么同一个类中的所有函数都将可以获得这个信息。

    注意

    • 找出只被赋值一次的临时变量

    动机

    • 表达式有可能非常复杂而难以阅读。临时变量可以帮助你将表达式分解
    • 在较长算法中，可以运用临时变量来解释每一步运算的意义

    使用

    • 在Extract Method 需要花费更大工作量的时候使用

    Split Temporary Variable（分解临时变量）

    针对每次赋值，创造一个独立、对应的临时变量。

    double temp = 2 * (_height + _width);
System.out.println(temp);
temp = _height * _width;
System.out.println(temp);

    final double perimeter = 2 * (_height + _width);
System.out.println(perimeter);
final double area = _height * _width;
System.out.println(area);

    动机

    • 临时变量应该只被赋值一次。如果超过一次，就意味着它们在函数中承担了一个以上的责任。

    Remove Assignments to Parameters（移除对参数的赋值）

    以一个临时变量取代该参数的位置

    int discount (int inputVal, int quantity , int yearToDate) {
    if (inputVal > 50 ) input -= 2;

    int discount(int inputVal,int quantity,int yearToDate) {
    int result = inputVal;
    if (inputVal > 50 ) result -= 2;

    动机

    • 这样的做法，降低了代码的清晰度，而且混用了按值传递和按引用传递这两种参数传递方式

    Replace Method with Method Object （以函数对象取代函数）

    将这个函数放进一个单独对象中，如此一来局部变量就成了对象内的字段

    class Order {
    double price() {
        double primaryBasePrice;
        double secondaryBasePrice;
        double tertiaryBasePrice;
    }
}

    动机

    • 局部变量的存在会增加函数分解难度
    • 无法用Replace Temp with Query 拆解一个需要拆解的函数

    Substitute Algorithm （替换算法）

    将函数本体替换为另一个算法

    String foundPerson(String[] people){
    for (int i =0; i <people.length;i++){
        if (people[0].equals("Don")){
            return "Don";
        }
    }
    return "";
}

    String foundPerson(String[] people){
    List candidates = Arrays.asList(new String[]{"Don","John"});
    for (int i =0; i <people.length;i++){
        if (candidates.contains(people[i])){
            return people[i];
        }
    }
    return "";
}
